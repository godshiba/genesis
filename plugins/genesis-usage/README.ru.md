# GENESIS Usage Sensor — сенсор лимитов

**Сенсор, а не HUD.** Он молчит, пока вы не подойдёте вплотную к лимиту Claude
Code, и тогда подсказывает сохраниться — или, если попросите, делает это сам.
Это вторая половина аварийного закрытия сессии G7 из GENESIS.

Хорошие HUD-ы для лимитов уже есть (claude-hud, cc-usage-monitor). Это не ещё
один такой. Он отвечает на другой вопрос: не «сколько я уже израсходовал?», а
«…значит, заверши сессию начисто — прямо сейчас, пока обрыв не бросил работу на
полуслове».

---

## Впервые здесь? Версия на 30 секунд

1. Установка (три строки, выполняйте по одной):

   ```text
   /plugin marketplace add godshiba/genesis
   /plugin install genesis-usage@godshiba
   /genesis-usage:setup
   ```

2. Всё. Теперь, когда вы за работой подойдёте близко к 5-часовому или недельному
   лимиту, Claude предупредит и посоветует запустить `/genesis:close`, чтобы вы
   не потеряли место, на котором остановились. Остальная часть страницы — для
   тонкой настройки.

**Одна честная оговорка:** данные о лимите есть только у подписчиков
**Claude.ai Pro/Max** (Claude Code v1.0.80+) и только **после первого ответа
Claude в сессии** — именно тогда Claude Code начинает присылать `rate_limits`.
На оплате по API-ключу данных о лимите нет, поэтому сенсор молчит — это так и
задумано, а не баг.

---

## Что он на самом деле делает

Claude Code присылает на stdin хука подписочный объект `rate_limits` — окна
`five_hour` и `seven_day`, у каждого `used_percentage` и `resets_at`. Хук
**Stop** (срабатывает по завершении каждого хода) читает их и решает, говорить
ли:

- **advise** (по умолчанию) — печатает однострочную рекомендацию в терминал и
  даёт ходу завершиться как обычно.
- **enforce** — блокирует ход, чтобы Claude запустил `/genesis:close` перед
  завершением. Только внутри проекта GENESIS; в остальных случаях откатывается
  к advise.

Срабатывает **не чаще одного раза на каждый новый 5-балльный шаг расхода за
сессию** и только когда до сброса окна ещё достаточно далеко, чтобы реально
рисковать обрывом, — подсказка в нужный момент, а не нытьё каждый ход.

```
5h usage 91% (resets in 0h23m). Run /genesis:close now - capture mid-task
state in SESSION_LOG before the cutoff.
```

## Конфигурация

`/genesis-usage:setup` — простой путь: он спросит, что вам нужно, и сам всё
запишет. `/genesis-usage:setup 80` сразу ставит 5-часовой порог на 80.

Хотите вручную или нужен полный список? Каждая опция — переменная окружения.
Задавайте их **глобально** в `~/.claude/settings.json` в блоке `"env"` или **на
уровне проекта** в его `.claude/settings.json` (там значение переопределяет
глобальное — то, что нужно, когда строгий порог требуется лишь некоторым
репозиториям):

| Переменная | По умолчанию | Что делает |
|------------|--------------|------------|
| `GENESIS_USAGE_MODE` | `advise` | `advise` (предупредить), `enforce` (блокировать, чтобы Claude закрыл сессию) или `off`. |
| `GENESIS_USAGE_THRESHOLD` | `90` | 5-часовой порог расхода (%), при котором срабатывает. Поставьте `80`, чтобы раньше. |
| `GENESIS_USAGE_WEEK_THRESHOLD` | `85` | Недельный порог расхода (%). |
| `GENESIS_USAGE_GRACE_SECS` | `120` | Молчать, если окно сбрасывается в пределах этого числа секунд (оно вот-вот обновится). |
| `GENESIS_USAGE_NOTIFY` | `off` | `on` также шлёт уведомление macOS при срабатывании сенсора. |
| `GENESIS_OFF` | не задано | `1` полностью отключает этот и все хуки GENESIS. |

Пример — срабатывать на 80% и пинговать, даже когда я в другом приложении:

```json
{
  "env": {
    "GENESIS_USAGE_THRESHOLD": "80",
    "GENESIS_USAGE_NOTIFY": "on"
  }
}
```

Сенсор перечитывает окружение каждый ход, так что изменения вступают в силу со
следующего хода — перезапуск не нужен (единственное исключение — статусная
строка ниже).

## Уведомления на рабочий стол

Поставьте `GENESIS_USAGE_NOTIFY=on` — и при срабатывании сенсора вы получите
нативное уведомление macOS, так что узнаете о близком лимите, даже когда Claude
Code спрятан за браузером или редактором. Используется `osascript`; где его нет,
уведомление просто пропускается, а предупреждение в терминале всё равно
появляется.

Это лёгкий ответ на «а если я отойду» — без всякого приложения в строке меню.

## Необязательная статусная строка

Краткая однострочная сводка — `<model>  ctx 38%  5h 91% (23m)  7d 62% (3d4h)  <branch>`
— с расходом в цветах зелёный/жёлтый/красный и тусклым обратным отсчётом после
каждой цифры: через сколько это окно обновится. Первое поле — модель, которую
вы запустили; последнее — текущая ветка git. Она **по желанию** (активной может
быть только одна статусная строка, поэтому плагин никогда не перетирает вашу).
`/genesis-usage:setup` её подключит. Намеренно минимальна: без разбора
инструментов/агентов/задач, без рамок. Нужен полноценный HUD — запускайте
отдельный HUD-плагин рядом с этим сенсором.

## Если что-то не так

- **Я вообще ничего не вижу.** Скорее всего, всё работает как надо. Проверьте по
  порядку: у вас Pro/Max (а не API-ключ)? Claude уже ответил хотя бы раз в этой
  сессии? Вы действительно выше порога? Не стоит ли `GENESIS_USAGE_MODE` в `off`
  или `GENESIS_OFF=1`?
- **Предупредил один раз и затих.** Так и задумано — срабатывает раз на каждый
  новый 5-балльный шаг за сессию, поэтому заговорит снова на следующем шаге
  (например, 90 → 95), а не каждый ход.
- **Предупредил прямо перед сбросом лимита.** Поднимите
  `GENESIS_USAGE_GRACE_SECS`, чтобы он молчал, когда сброс уже близко.
- **Режим enforce не заблокировал.** enforce блокирует только внутри проекта
  GENESIS (где есть `docs/registry/`); в остальных случаях — советует.

## Безопасный отказ и приватность

Молчит без `python3`, без `rate_limits` (пользователи API-ключа) или при любой
ошибке разбора — в режиме `advise` никогда не блокирует ход и никогда не
блокирует вне проекта GENESIS. Читает только то, что Claude Code и так передаёт
на stdin, не делает сетевых запросов и пишет лишь крошечный файл-троттлинг во
временный каталог.

---

## Связь с GENESIS

У `/genesis:close` из плагина [`genesis`](../genesis) есть аварийный режим,
который фиксирует состояние посреди задачи перед обрывом. Этот сенсор — то, что
подсказывает вам (или Claude) запустить его в нужный момент. Полезен и сам по
себе; вместе — лучше.
