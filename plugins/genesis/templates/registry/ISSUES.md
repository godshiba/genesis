# ISSUES.md — Open Issues and Deferred Work

Canonical register. **G3: every bug, deferral, or known limitation gets an
entry.** Sequential IDs, severity-tagged, with concrete investigation paths.
SESSION_LOG references issues by ID; the body lives here only.

Severities: `high` (blocks or corrupts), `medium` (wrong but survivable),
`deferred` (intentional, revisit later).

If the project has a GitHub remote, mirror entries with
`gh issue create --title "ISS-NNN: <title>"` and back-fill the link line.
Resolve both in the same commit — they must not drift.
