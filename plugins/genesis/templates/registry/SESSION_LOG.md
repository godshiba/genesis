# SESSION_LOG.md — What Happened, When, and What Is Next

**G7: append an entry before ending any work session.** Newest entries on top.
Be concrete: file paths, what changed, why. Reference issues by ID (ISS-NNN) —
their bodies live in ISSUES.md only.

**Resume protocol:** the next session reads the top entry here, then
ROADMAP.md's first unchecked task, and continues. Write every entry for that
reader.

A closing entry must include:
1. What was completed.
2. Repo state (branch, last commit, check command green or not).
3. The next concrete step.
4. Anything in progress that should be reviewed first.
